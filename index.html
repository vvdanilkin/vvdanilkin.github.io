Проекты

1. Тестирование веб- и мобильного приложения Яндекс.Самокат


Дипломный проект по тестированию учебного сервиса по заказу и доставке самокатов в Москве. Сервис состоит из веб-приложения для клиентов и мобильного приложения для курьеров, доставляющих заказнные самокаты. В ходе проекта с помощью применения инструментов DevTools, Figma, Charles, Postman, PostgreSQL, Cygwin удалось проработать и протестировать следующие аспекты:


  • - Mindmap функциональности формы заказа
  • - Чек-лист по требованиям к функциональности "Статус заказа"
  • - Чек-лист валидации полей формы заказа
  • - Тест-кейсы на проверку функциональности мобильного приложения
  • - Чек-лист на проверку API веб-приложения
  • - Баг-репорты

2. Работа с консолью


Необходимо было изучить логи и отобрать те, которые исходят от определенного IP-адреса, а также разложить логи с различными кодами ошибок по разным файлам


  • - Команды для работы с консолью

3. Работа с базами данных


В реляционной базе данных, состоящей из нескольких таблиц, необходимо было с помощью инструмента PostgreSQL составить запросы для выборки количества совершенных поездок, количества имеющихся в каждой компании автомобилей, данные о погоде за указанный период


  • - SQL-запросы для составления необходимых выборок

4. Тестирование мобильного приложения Яндекс.Метро


Необходимо было проанализировать обозначенную часть требований, составить чек-лист и провести тестирование мобильного приложения Яндекс.Метро с использованием реального устройства или устройства Android Studio.


  • - Чек-лист к функциональным требованиям

5. Тестирование API веб-приложения Яндекс.Прилавок


Необходимо было составить чек-лист и провести тестирование API веб-приложения Яндекс.Прилавок с использованием инструмента Postman


  • - Чек-лист к функциональным требованиям

6. Тестирование веб-приложения Яндекс.Маршруты


Тестированию учебного сервиса по построению маршрутов для разных видов транспорта с возможностью заказа такси и каршеринга. С помощью применения инструментов DevTools, Figma, Charles удалось проработать и протестировать следующие аспекты:


  • - Чек-лист тестирования окон «Способ оплаты» и «Добавление карты»
  • - Чек-лист тестирования вёрстки формы бронирования каршеринга
  • - Тест-кейсы и тестирование логики функционирования кнопки «Забронировать»
  • - Тест-кейсы и тестирование логики функциональности бронирования
  • - Чек-лист на добавление нового вида транспорта "Аэротакси"
  • - Классы эквивалентности и граничные значения для полей ввода времени и адреса
  • - Тест-кейсы функциональности расчета стоимости и продолжительности поездки на каршеринге
  • - Блок-схема логики выбора скорости для расчета стоимости и продолжительности поездки
  • - Mindmap интерфейса и логики работы приложения
